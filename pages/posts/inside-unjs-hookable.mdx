[hookable](https://github.com/unjs/hookable) 為 [UnJS](https://github.com/unjs) 團隊釋出的套件，可以簡單想像成 event emitter

共通概念如下：

- 定義「事件」與「觀察者」之間的一對多關係
- 事件由一個名稱及對應的資料格式組成，不同事件都有其對應的觀察者介面
- 事件發生時，向已註冊的觀察者推送資料
- 事件只關注觀察者的介面，不會知道觀察者的實作細節，藉此保持鬆耦合
- 觀察者依據不同事件的介面實作監聽方法

主要差異：

- hookable 透過 `callHook` 觸發 listeners，預設以非同步方式依序執行 ([下方說明](#awaitable-hooks))
- 為了讓非同步的 log 更好被追蹤，在調用 listeners 前，會透過 [console.createTask](https://developer.chrome.com/docs/devtools/console/api#createtask) 建立關聯，可參考 [pull#69](https://github.com/unjs/hookable/pull/69)
- 上述兩點需要額外的開銷，如果沒有非同步需求，且有效能考量，event emitter 或許是比較好的選擇，可參考 [issue#64](https://github.com/unjs/hookable/issues/64#issuecomment-1430076522)

## Internal data structure

- 透過一個名為 `_hooks` 的 internal object 來儲存 listeners
- key 為 hook name
- value 為已註冊的 listeners array

```ts
class Hookable<
  HooksT extends Record<string, any> = Record<string, HookCallback>,
  HookNameT extends HookKeys<HooksT> = HookKeys<HooksT>
> {
  private _hooks: { [key: string]: HookCallback[] };
  // skip

  constructor() {
    this._hooks = {};
    // skip
  }

  hook<NameT extends HookNameT>(
    name: NameT,
    function_: InferCallback<HooksT, NameT>,
    options: { allowDeprecated?: boolean } = {}
  ) {
    // skip
    this._hooks[name] = this._hooks[name] || [];
    this._hooks[name].push(function_);
    // skip
  }
  // skip
}
```

大致示意如下：

```ts
const hooks = hookable.createHooks();
hooks.hook('hello:foo', () => {});
hooks.hook('hello:bar', () => {});

/**
 * _hooks
 *
 * {
 *   'hello:foo': [() => {}],
 *   'hello:bar': [() => {}],
 * }
*/

```

## Awaitable hooks

- listeners 可以為 promise based 的 asynchronous function
- `await callHook{:js}` 可以確保對應的 listeners 皆已執行完成

```ts
const hooks = createHooks();
hooks.hook('hello', foo);
hooks.hook('hello', bar);

function foo() {
  return new Promise((resolve) => {
    console.log('foo started');
    setTimeout(() => {
      console.log('foo resolved');
      resolve();
    }, 200);
  });
}

function bar() {
  return new Promise((resolve) => {
    console.log('bar started');
    setTimeout(() => {
      console.log('bar resolved');
      resolve();
    }, 200);
  });
}

async function run() {
  await hooks.callHook('hello');
  console.log('hook finished');
}

run();
// "foo started"
// "foo resolved"
// "bar started"
// "bar resolved"
// "hook finished"
```

`callHook` 預設使用 `serialTaskCaller`

下方為 `serialTaskCaller` 的程式碼，這裡附上 [source code](https://github.com/unjs/hookable/blob/ddc01ba16b9d66da16f40e995cc2e63970386727/src/utils.ts#L67-L76)

```ts showLineNumbers
function serialTaskCaller(hooks: HookCallback[], args: any[]) {
  const name = args.shift();
  const task = createTask(name);
  // eslint-disable-next-line unicorn/no-array-reduce
  return hooks.reduce(
    (promise, hookFunction) =>
      promise.then(() => task.run(() => hookFunction(...args))),
    Promise.resolve()
  );
}
```

第 3 行的 `const task = createTask(name);{:js}` 與 [console.createTask](https://developer.chrome.com/docs/devtools/console/api#createtask) 兼容相關

第 5-9 行

1. 第一次調用 listener，是透過 `Promise.resolve{:js}` 產生的 promise
1. 後續透過 `Promise.then{:js}` 產生新的 promise 並委派 fulfilled callback，當 promise 狀態為 fulfilled 才會調用下一個 listener
1. 回傳最後一個 listener 對應的 promise，當最後一個 promise 的狀態變更成 fulfilled，即代表所有 listeners 已經依序執行完成
